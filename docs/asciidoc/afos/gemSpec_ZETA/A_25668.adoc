==== A_25668 - PEP HTTP Proxy - Access Token Validierung

Die Komponente HTTP Proxy MUSS das übergebene Access Token validieren.
Insbesondere MÜSSEN

* die Signatur des Authorization Servers gültig,
* die Angaben zur zeitlichen Gültigkeit (Felder: iat, exp) valide,

* die Angabe aud für das Resource Backend korrekt eingetragen und
* die Angabe scope und aud passend zur Request url sein.
Die Signatur des Access Tokens ist gültig, wenn sie mathematisch gültig ist und die Signatur vom Authorization Server im gleichen ZETA Guard erstellt wurde (default Einstellung) oder von einem Authorization Server, der in der Konfiguration des HTTP Proxy angegeben und im Entity Statement des Federation Master aufgeführt ist.
Es MUSS möglich sein, mehrere Authorization Server in die Konfiguration des HTTP Proxy einzutragen.
Wenn das Access Tokens ungültig ist, dann MUSS der Request mit HTTP Code 401 Unauthorized beantwortet werden.
