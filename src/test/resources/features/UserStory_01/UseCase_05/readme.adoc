== UseCase_01_05: Client registrierung stationaer SC 409

.UseCase_01_05
[cols="1,1a",options="autowidth"]
|===
| *Titel:* | Client registrierung stationaer SC 409

| Akteur: |
*MA einer LEI*

| weitere Akteure: |
- *ZETA Client*
- *PDP Authorization Server*

| Auslöser: |
Ein MA einer LEI versucht, einen stationären ZETA Client zu registrieren, obwohl bereits eine Registrierung für diesen Client existiert.

| Kurzbeschreibung: |
Der ZETA Client sendet eine Dynamic Client Registration an den PDP Authorization Server.
Aufgrund eines bestehenden Konflikts (z. B. bereits registrierter Client) lehnt der PDP Authorization Server die Anfrage mit ``409 Conflict`` ab.
Der ZETA Client informiert den MA, dass keine weitere Registrierung möglich ist.

| Vorbedingung: |
Der stationäre ZETA Client ist bereits beim ZETA Guard registriert oder es existiert ein Konflikt mit einer bestehenden Registrierung.

| Nebenbedingung: |
Keine zusätzlichen Nebenbedingungen.

| Ablauf: |
. Der MA startet die *initiale Registrierung* im ZETA Client.
. Der ZETA Client generiert ein *Client Instance Key Pair*.
. Der ZETA Client sendet eine *Dynamic Client Registration (DCR)* an den PDP Authorization Server (``POST /register``).
. Der PDP Authorization Server erkennt einen Konflikt (z. B. bereits vorhandene Registrierung).
. Der PDP Authorization Server antwortet mit ``409 Conflict``.
. Der ZETA Client informiert den MA, dass der Client bereits registriert ist und keine weitere Registrierung erfolgen darf.

| Nachbedingung: |
Es wurde keine neue Registrierung erzeugt.
Die bestehende Registrierung bleibt unverändert.
