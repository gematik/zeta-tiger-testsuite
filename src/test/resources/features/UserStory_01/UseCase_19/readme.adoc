[#UC_01_19]
== UseCase_01_19: Client ressource anfrage fachdienst SC 405

.UseCase_01_19
[cols="1,1a",options="autowidth"]
|===
| *Titel:* | Client ressource anfrage fachdienst SC 405

| Akteur: |
*MA einer LEI*

| weitere Akteure: |
- *ZETA Client*
- *PEP HTTP Proxy*

| Auslöser: |
Ein MA einer LEI fordert eine Ressource über den ZETA Client an, verwendet aber eine für den Endpunkt nicht erlaubte HTTP Methode.

| Kurzbeschreibung: |
Der ZETA Client sendet eine Anfrage mit gültigem Access Token und DPoP Nachweis an den PEP HTTP Proxy.
Der Endpunkt unterstützt die verwendete HTTP Methode nicht, daher liefert der PEP HTTP Proxy
``405 Method Not Allowed`` und verweigert den Zugriff.

| Vorbedingung: |
Der ZETA Client ist registriert, authentisiert und autorisiert.

| Nebenbedingung: |
Der PEP HTTP Proxy ist so konfiguriert, dass der Endpunkt nur eine bestimmte HTTP Methode (z. B. ``GET``) erlaubt.

| Ablauf: |
1. MA startet den Abruf einer geschützten Ressource über den ZETA Client mit einer nicht erlaubten HTTP Methode.
2. Der ZETA Client erstellt einen *DPoP Nachweis*.
3. Der ZETA Client ruft die Ressource am PEP HTTP Proxy auf:
- ``POST /resource`` (Beispiel für nicht erlaubte Methode)
- ``Authorization: DPoP ... (Access Token)``
- ``DPoP: ... (DPoP Token)``
4. Der PEP HTTP Proxy validiert die Nachweise und prüft die HTTP Methode.
5. Der PEP HTTP Proxy antwortet mit ``405 Method Not Allowed``.
6. Der ZETA Client informiert den MA, dass die Anfrage mit der gewählten HTTP Methode nicht unterstützt wird.

| Nachbedingung: |
Der Ressourcenzugriff wurde abgewiesen.

|===
