[#UC_01_21]
== UseCase_01_21: Client ressource anfrage fachdienst SC 504

.UseCase_01_21
[cols="1,1a",options="autowidth"]
|===
| *Titel:* | Client ressource anfrage fachdienst SC 504

| Akteur: |
*MA einer LEI*

| weitere Akteure: |
- *ZETA Client*
- *PEP HTTP Proxy*
- *Resource Server*

| Auslöser: |
Ein MA einer LEI fordert eine Ressource über den ZETA Client an.

| Kurzbeschreibung: |
Der ZETA Client sendet eine Anfrage mit einem Access Token für eine geschützte Ressource an den PEP HTTP Proxy.
Der PEP HTTP Proxy leitet die Anfrage an den Resource Server weiter, erhält jedoch innerhalb des
konfigurierten Timeouts keine Antwort. +
Der PEP HTTP Proxy antwortet deshalb mit ``504 Gateway Timeout``.
Der ZETA Client informiert den MA und empfiehlt,
die Anfrage nach einer angemessenen Wartezeit erneut zu versuchen.

| Vorbedingung: |
Der ZETA Client ist registriert, authentisiert und autorisiert.

| Nebenbedingung: |
Der Resource Server antwortet nicht innerhalb des vom PEP HTTP Proxy konfigurierten Timeouts.

| Ablauf: |
. MA startet den Abruf einer geschützten Ressource über den ZETA Client.
. Der ZETA Client erstellt einen *DPoP Nachweis*.
. Der ZETA Client ruft die Ressource am PEP HTTP Proxy auf:
- ``GET /resource``
- ``Authorization: DPoP ... (Access Token)``
- ``DPoP: ... (DPoP Token)``
. Der PEP HTTP Proxy prüft das *Access Token*.
. Der PEP HTTP Proxy prüft den *DPoP Nachweis*.
. Der PEP HTTP Proxy leitet die Anfrage an den Resource Server weiter.
. Der Resource Server antwortet nicht innerhalb des konfigurierten Timeouts.
. Der PEP HTTP Proxy antwortet mit ``504 Gateway Timeout``.
. Der ZETA Client informiert den MA über die Zeitüberschreitung und empfiehlt einen erneuten Versuch nach angemessener Wartezeit.

| Nachbedingung: |
Die Ressource wurde nicht geliefert.
Der MA ist informiert und kann die Anfrage nach einer angemessenen Wartezeit erneut stellen.
