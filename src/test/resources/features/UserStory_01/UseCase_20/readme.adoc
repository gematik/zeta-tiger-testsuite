[#UC_01_20]
== UseCase_01_20: Client ressource anfrage fachdienst SC 502

.UseCase_01_20
[cols="1,1a",options="autowidth"]
|===
| *Titel:* | Client ressource anfrage fachdienst SC 502

| Akteur: |
*MA einer LEI*

| weitere Akteure: |
- *ZETA Client*
- *PEP HTTP Proxy*
- *Resource Server*

| Auslöser: |
Ein MA einer LEI fordert eine Ressource über den ZETA Client an.

| Kurzbeschreibung: |
Der ZETA Client sendet eine Anfrage mit einem Access Token für eine geschützte Ressource an den PEP HTTP Proxy.
Der PEP HTTP Proxy leitet die Anfrage an den Resource Server weiter, erhält jedoch eine ungültige Antwort. +
Der PEP HTTP Proxy antwortet deshalb mit ``502 Bad Gateway``.
Der ZETA Client informiert den MA und empfiehlt,
die Anfrage später erneut zu versuchen.

| Vorbedingung: |
Der ZETA Client ist registriert, authentisiert und autorisiert.

| Nebenbedingung: |
Der Resource Server liefert eine ungültige Antwort auf die Anfrage.

| Ablauf: |
. MA startet den Abruf einer geschützten Ressource über den ZETA Client.
. Der ZETA Client erstellt einen *DPoP Nachweis*.
. Der ZETA Client ruft die Ressource am PEP HTTP Proxy auf:
- ``GET /resource``
- ``Authorization: DPoP ... (Access Token)``
- ``DPoP: ... (DPoP Token)``
. Der PEP HTTP Proxy prüft das *Access Token*.
. Der PEP HTTP Proxy prüft den *DPoP Nachweis*.
. Der PEP HTTP Proxy leitet die Anfrage an den Resource Server weiter.
. Der Resource Server liefert eine ungültige Antwort.
. Der PEP HTTP Proxy antwortet mit ``502 Bad Gateway``.
. Der ZETA Client informiert den MA über die fehlerhafte Antwort und weist auf einen erneuten Versuch zu einem späteren Zeitpunkt hin.

| Nachbedingung: |
Die Ressource wurde nicht geliefert.
Der MA ist informiert und kann die Anfrage zu einem späteren Zeitpunkt erneut stellen.
