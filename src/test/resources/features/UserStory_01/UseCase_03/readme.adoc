== UseCase_01_03: Client registrierung stationaer SC 401

.UseCase_01_03
[cols="1,1a",options="autowidth"]
|===
| *Titel:* | Client registrierung stationaer SC 401

| Akteur: |
*MA einer LEI*

| weitere Akteure: |
- *ZETA Client*
- *PDP Authorization Server*

| Auslöser: |
Ein MA einer LEI startet die Registrierung eines stationären ZETA Clients, die notwendige Authentifizierung für die Registrierung ist jedoch ungültig oder fehlt.

| Kurzbeschreibung: |
Der ZETA Client sendet eine Dynamic Client Registration an den PDP Authorization Server, ohne die erforderliche
Authentifizierung korrekt nachzuweisen. Der PDP Authorization Server weist die Anfrage mit ``401 Unauthorized`` zurück.
Der ZETA Client informiert den MA über die fehlgeschlagene Registrierung und fordert eine erneute Authentifizierung.

| Vorbedingung: |
Der stationäre ZETA Client ist noch nicht beim ZETA Guard registriert.

| Nebenbedingung: |
Keine zusätzlichen Nebenbedingungen.

| Ablauf: |
. Der MA startet die *initiale Registrierung* im ZETA Client.
. Der ZETA Client generiert ein *Client Instance Key Pair*.
. Der ZETA Client sendet eine *Dynamic Client Registration (DCR)* an den PDP Authorization Server (``POST /register``),
   jedoch ohne gültige Authentisierungsnachweise.
. Der PDP Authorization Server prüft die Authentisierung der Registrierung.
. Der PDP Authorization Server antwortet mit ``401 Unauthorized``.
. Der ZETA Client informiert den MA, dass eine erneute Authentisierung erforderlich ist.

| Nachbedingung: |
Der stationäre ZETA Client besitzt keine ``client_id``. Es wurde keine Registrierung in der PDP Datenbank gespeichert.
