[#UC_01_26]
== UseCase_01_26: Client ressource anfrage fachdienst SC 500

.UseCase_01_26
[cols="1,1a",options="autowidth"]
|===
| *Titel:* | Client ressource anfrage fachdienst SC 500

| Akteur: |
*MA einer LEI*

| weitere Akteure: |
- *ZETA Client*
- *PEP HTTP Proxy*
- *Resource Server*

| Auslöser: |
Ein MA einer LEI fordert eine Ressource über den ZETA Client an.

| Kurzbeschreibung: |
Der ZETA Client sendet eine Anfrage mit einem Access Token für eine geschützte Ressource an den PEP HTTP Proxy.
Der PEP HTTP Proxy leitet die Anfrage an den Resource Server weiter, erhält jedoch eine Fehlerantwort mit ``ZETA-Cause: Proxy``. +
Der PEP HTTP Proxy wertet dies als Fehler des HTTP Proxy, leitet die Antwort nicht an den Client weiter
und antwortet mit ``500 Internal Server Error: Allgemeiner Serverfehler. Der Client sollte den Vorgang möglicherweise zu einem späteren Zeitpunkt wiederholen oder den Nutzer auf ein Problem auf dem Server hinweisen.``.
Der ZETA Client informiert den MA und empfiehlt,
die Anfrage nach einer angemessenen Wartezeit erneut zu versuchen oder auf ein Serverproblem hinzuweisen.

| Vorbedingung: |
Der ZETA Client ist registriert, authentisiert und autorisiert.

| Nebenbedingung: |
Der Resource Server liefert eine Fehlerantwort mit ``ZETA-Cause: Proxy``.

| Ablauf: |
. MA startet den Abruf einer geschützten Ressource über den ZETA Client.
. Der ZETA Client erstellt einen *DPoP Nachweis*.
. Der ZETA Client ruft die Ressource am PEP HTTP Proxy auf:
- ``GET /resource``
- ``Authorization: DPoP ... (Access Token)``
- ``DPoP: ... (DPoP Token)``
. Der PEP HTTP Proxy prüft das *Access Token*.
. Der PEP HTTP Proxy prüft den *DPoP Nachweis*.
. Der PEP HTTP Proxy leitet die Anfrage an den Resource Server weiter.
. Der Resource Server antwortet mit Fehlercode ``500 Internal Server Error``, einer Fehlerantwort und setzt ``ZETA-Cause: Proxy``.
. Der PEP HTTP Proxy verwirft die Antwort vom Resource Server und antwortet mit ``500 Internal Server Error``.
. Der ZETA Client weist auf einen allgemeinen Serverfehler hin und informiert den MA, dass der Vorgang möglicherweise zu einem späteren Zeitpunkt wiederholt werden sollte oder ein Problem auf dem Server vorliegt.

| Nachbedingung: |
Die Ressource wurde nicht geliefert.
Der MA ist informiert und kann die Anfrage nach einer angemessenen Wartezeit erneut stellen.
