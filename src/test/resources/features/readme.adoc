== Definierte User Stories

Es wurden auf Basis der zu testenden Anforderung einige User-Stories definiert, welche Use-Cases beinhalten, denen wiederum Testaspekte zugeordnet werden können.
Die textuellen Beschreibungen der User Stories und Use Cases werden im Testplan standardmäßig vollständig dargestellt.
Die eingebetteten ``.feature``-Dateien werden nur gerendert, wenn das Asciidoctor-Attribut ``render_features`` gesetzt wird (z. B. ``-Arender_features``).

In Use-Case-Diagrammen werden ``critical``-Boxen verwendet.
Diese Boxen haben keine semantische Bedeutung.
Sie dienen nur zur besseren Abgrenzung funktionaler Sequenzen.
